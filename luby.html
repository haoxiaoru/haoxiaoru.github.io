Algorithm Analysis Report

Convergence of Luby's Algorithm on Cycle Graphs

Prepared by

Theoretical Computer Science Division

Date

June 2023

Classification

Randomized Algorithms

This report provides a theoretical analysis of the convergence rate of Luby's algorithm for finding maximal independent sets on cycle graphs. All results are based on established research in distributed computing and randomized algorithms.

Summary

This report analyzes the convergence rate of Luby's algorithm for finding a maximal independent set (MIS) on cycle graphs with n vertices. Luby's algorithm is a randomized parallel algorithm that operates in synchronous rounds, where vertices are probabilistically selected to join the MIS. For an n-vertex cycle graph, which has a constant degree (Δ=2), the algorithm converges in O(log n) rounds with high probability. Specifically, there exists a constant c such that after c·log n rounds, the algorithm terminates with probability at least 1-1/n. This logarithmic convergence rate is optimal for parallel MIS algorithms on sparse graphs and demonstrates the algorithm's efficiency on cycle topologies.

Key Takeaways

  • Asymptotic Convergence Rate: The function f₁(n) = Θ(log n) describes the number of rounds needed for Luby's algorithm to terminate on n-vertex cycle graphs with probability 1-1/n.
  • Complexity Classification: The algorithm's convergence rate falls into the logarithmic time complexity class, which is significantly more efficient than polynomial time algorithms.
  • Geometric Reduction: In each round, a constant fraction of remaining vertices is eliminated (either by joining the MIS or being adjacent to selected vertices), leading to a geometric decrease in problem size.
  • Sparsity Advantage: The cycle graph's constant degree (Δ=2) allows for efficient parallel execution, as the general bound O(log n · log Δ) simplifies to O(log n).
  • Probabilistic Guarantee: The algorithm is Las Vegas type - it always produces a correct MIS, but with randomized runtime that is O(log n) with high probability (1-1/n).
  • Parallel Efficiency: The logarithmic convergence rate demonstrates that MIS problems on cycles are highly parallelizable, requiring significantly fewer than n steps to solve.

Luby's Parallel MIS Algorithm

Luby's algorithm is a classic parallel randomized algorithm designed to find a maximal independent set (MIS) of a graph. It operates through a series of synchronous rounds where active vertices make probabilistic decisions about joining the MIS.

Algorithm Steps

1. Random Choice

Each active vertex selects a random priority value uniformly at random.

2. Selection Rule

A vertex joins the MIS if its priority exceeds all of its neighbors' priorities.

3. Removal

Selected vertices and their neighbors are removed from further consideration.

4. Iteration

The algorithm repeats on the remaining subgraph until no vertices remain.

"Luby's algorithm was a breakthrough result showing MIS can be computed in O(log n) time on a PRAM model." — Based on Luby's 1986 paper "A Simple Parallel Algorithm for the Maximal Independent Set Problem"

Convergence Analysis

The general theoretical result for Luby's algorithm states that on an n-vertex graph with maximum degree Δ, the algorithm terminates in O(log n · log Δ) rounds with high probability. For bounded-degree graphs where Δ is constant, this simplifies to O(log n) rounds.

Application to Cycle Graphs

In a cycle graph Cn, each vertex has exactly two neighbors (Δ = 2). Applying the general bound:

f₁(n) = O(log n · log 2) = O(log n)

Therefore, on a cycle of size n, Luby's MIS algorithm converges in time proportional to log n with high probability. More precisely, there exists a constant c such that after c·log₂ n parallel steps, the algorithm has produced an MIS with probability at least 1-1/n.

Cycle Graph Specifics

The cycle graph Cn is a particularly simple topology: n vertices arranged in a ring, where each vertex is connected to exactly two neighbors. This sparse structure allows for efficient execution of Luby's algorithm.

Key Properties of Cycle Graphs

  • Regular structure: Every vertex has exactly two neighbors (degree = 2)
  • Sparsity: Total number of edges is exactly n
  • Connectivity: Removing any single vertex breaks the cycle into a path
  • MIS size: Any MIS of a cycle with n vertices contains approximately n/3 vertices
"The cycle graph's sparsity ensures that Luby's MIS algorithm runs in O(log n) rounds with high probability." — Theoretical analysis of parallel algorithms on sparse graphs

Explanation of the O(log n) Bound for a Cycle

The logarithmic convergence rate of Luby's algorithm on cycle graphs can be understood through several key insights:

Constant Selection Probability

On a cycle, each vertex has exactly 2 neighbors. When all active vertices choose random priorities, any given vertex has a reasonable chance of having the highest priority in its local neighborhood (itself and its two neighbors). Specifically, the probability that a particular vertex is locally maximal is approximately 1/3, as there are three participants in each local comparison.

Geometric Reduction of Active Vertices

When a vertex is added to the MIS, it and its two neighbors become inactive (removed from consideration). This means that selecting one vertex effectively covers a length-3 segment of the cycle. Even though random choices are not independent, analysis shows that a constant fraction of vertices are removed in each round with high probability.

Logarithmic Number of Rounds

If in each round the remaining problem size drops by a constant factor (e.g., 1/2 or 2/3) on average, then after t rounds, the expected number of remaining vertices is approximately n·(1/2)ᵗ. After O(log n) rounds, the expected number of remaining vertices approaches zero.

By applying concentration bounds (such as Chernoff bounds), one can rigorously show that after O(log n) rounds, with probability at least 1-1/n, no vertices remain active.

Complexity Comparison

The O(log n) convergence rate for Luby's algorithm on cycle graphs is significantly more efficient than many alternative approaches:

Algorithm/Approach Time Complexity Comparison to Luby's
Sequential Greedy MIS O(n) Much slower for large n
Luby's Algorithm (Cycle) O(log n) Baseline
Deterministic Parallel MIS O(log² n) Slower by log n factor

The logarithmic convergence rate places Luby's algorithm in the highly efficient polylogarithmic complexity class, making it suitable for large-scale parallel implementations.

Theoretical Foundation

The analysis of Luby's algorithm's convergence rate is built upon several fundamental theoretical results:

  • Luby's Original Analysis (1986): Proved an O(log n) round complexity (in expectation and with high probability) for MIS on general graphs[1]
  • Concentration Bounds: Chernoff bounds and other probabilistic tools demonstrate that the number of rounds exceeds c·log n with probability at most 1/n[2]
  • Edge-Centric Analysis: Shows that each edge on the cycle has a good probability of being "resolved" in each round[3]
  • Phase-Based Analysis: Groups rounds into phases and shows that the maximum degree or total number of edges drops exponentially fast[4]
"Almost always, Luby's MIS algorithm terminates in O(log Δ · log n) rounds... with a small probability ≤ 1/n it may run longer." — From theoretical analysis of randomized distributed algorithms
